# Changelog

All notable changes to this project will be documented in this file.

## [1.0.1] - 2024-01-XX

### Added
- Interactive setup process for project initialization
- Interactive feature addition with guided prompts
- Support for multiple state management options (BLoC, Riverpod, basic)
- Network layer integration with Dio
- Navigation setup with Go Router
- Local storage integration with Hive
- Comprehensive error handling and failure management
- Theme and styling infrastructure
- Dependency injection with GetIt
- Feature-based architecture with Clean Architecture principles
- Automatic dependency management
- File and folder creation utilities
- String casing utilities (PascalCase, camelCase)

### Changed
- Improved command-line interface with better user experience
- Enhanced interactive CLI with guided prompts for feature addition
- Enhanced template generation with more comprehensive code structure
- Better error messages and validation
- Made feature name optional in add command with interactive fallback

### Fixed
- Project name detection from pubspec.yaml
- Command execution error handling
- File creation and folder structure generation

## [1.0.0] - 2024-01-XX

### Added
- Initial release of flutter_scaffold CLI tool
- Basic init command for project initialization
- Add feature command for creating new features
- Core template generation system
- Basic Clean Architecture folder structure
- State management template generation
- Command-line argument parsing
- File utilities for project manipulation
